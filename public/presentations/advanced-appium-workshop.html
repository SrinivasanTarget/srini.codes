01 · FOUNDATION

Appium Philosophy

  • "You're testing the same app you are shipping"
  • Write tests in any language and framework you want
  • Wrapped vendor-provided frameworks in WebDriver API WebDriver Protocol
  • Fully open source
Appium aims to automate any mobile app from any language using any test framework, with full access to back-end APIs and databases from test code.

java WebElement slider = driver.findElement(AppiumBy.accessibilityId("slider")); Point source = slider.getLocation(); PointerInput finger = new PointerInput(PointerInput.Kind.TOUCH, "finger"); Sequence sequence = new Sequence(finger, 1); sequence.addAction(finger.createPointerMove( ofMillis(0), PointerInput.Origin.viewport(), source.x, source.y)); sequence.addAction(finger.createPointerDown( PointerInput.MouseButton.MIDDLE.asArg())); sequence.addAction(new Pause(finger, ofMillis(600))); sequence.addAction(finger.createPointerMove( ofMillis(600), PointerInput.Origin.viewport(), source.x + 400, source.y)); sequence.addAction(finger.createPointerUp( PointerInput.MouseButton.MIDDLE.asArg())); driver.perform(singletonList(sequence));

12 · PLUGINS

What is an Appium 3.0 Plugin?

  • Update the Appium server object before it starts listening for requests
  • Add arbitrary functionality that executes before or after actual Appium commands
  • Alter the Appium server to introduce new commands and distribute them
🧪

pluginE2EHarness

Configures server and driver for testing via before/after hooks

🔬

driverE2ETestSuite

Creates Mocha test suite making HTTP requests to in-memory server

20 · PLUGIN

Device Farm Plugin

bashappium plugin install --source=npm appium-device-farm appium --use-plugins=device-farm
  • Auto-detects connected Android, iOS Simulators, and Real devices
  • Dynamically allocates a free device from the pool
  • Updates device pool with new devices during test execution
  • Queues test cases when no free device is available
  • Allocates random ports for parallel execution
